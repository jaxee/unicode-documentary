What is Unicode?

Unicode is a well know character encoding standard developed by a group of companies and institutions called the Unicode Consortium, which consists of many big named companies like MathML, Adobe Systems Incorporated and Microsoft Corporation [Pournader, 2002, pg 80]

Unicode provides a unique number for every character, no matter what the platform, no matter what the program, no matter what the language

Unicode is the successor to the American Standard Code for Information Interchange (ASCII) which could only display Latin alphabet characters.

Compared to ASCII which could only display Latin alphabet characters, Unicode can store 65,536 different characters, the first 127 characters are the same as ASCII.

Why?

The main goal and interest was to develop an international text-encoder and application that can be used in all different countries and platforms. It solves the main problems that many current programs and encoders have: the fact that they are problems with recognizing the font and the use of multiple and inconsistent characters’ codes in many other languages other than English [BigeLow & Holmes, 1993, pg 289]

Where is it used?

The Unicode Standard has been adopted by such industry leaders as Apple, HP, IBM, JustSystems, Microsoft, Oracle, SAP, Sun, Sybase, Unisys and many others

It has been gaining momentum in recent years, especially in the software and computer industry [Tull, 2002]

It is supported in many operating systems, all modern browsers, and many other products

According to Google, it was reported that only 50% of all web pages were encoded in Unicode in January 2010. Today, over 70% of the most popular websites uses Unicode [John, 2013, pg 334]

Importance of Language

Writing Systems

A writing system is the interpretation of verbal communication in the form of different symbols. Humans use many forms of communication but writing can store information and share it with others. [Coulmas, 1989]

Writing systems can be placed in three very broad categories; alphabets, syllabaries, or logographics [Taylor & Olson, 1995]. Logographic systems are symbols that represent a word or phrase [Tzeng & Wang, 1983]. Syllabaries are systems where written symbols represent syllables [Tzeng & Wang, 1983]. Alphabet systems mean letters in the script correspond to a consonant or vowel segment in the spoken language [Tzeng & Wang, 1983].

Unicode is an effort to try to represent all reasonable writing systems on the planet:

  • It presents a completely different way of thinking about characters
  • Unicode labels each character with a code point which is usually represented in hexadecimal
  • The Unicode group mapped every single character in each language to a code point
  • These code points must be encoded and UTF-8 became the system to store the code point from 0-127 in a single byte and the rest of the stored using 2 plus bytes

Unicode Transformation Format

Unicode transformation format (UTF) is an algorithmic mapping from every Unicode code point (except surrogate code points) to a unique byte sequence. There are several possible representations of Unicode data, including UTF-8, UTF-16 and UTF-32. The most popular ones, however are UTF-8 and UTF-16

What are the main differences?

  • UTF-8 uses a minimum of one byte (8-bits) to encode characters
  • It is backward-compatible with ASCII
  • UTF-16 uses a minimum of two bytes/16 bits
  • It is incompatible with ASCII

UTF-16, UTF-8, and UCS-2 Encoding Examples

UTF-8

The main goal of UTF-8 is to encode Unicode characters in one byte.

At a high level, code points 0 to 7 are stored as regular, single byte ASCII. Code points 80 and above are converted to binary and stored in a series of bytes. The first “count” byte indicates the number of bytes for the code point, including the count byte.

Bytes starting with 10 are data bytes and contain information on the code point. There are no null bytes and characters outside of 0-127 start with a 1 to show that they are not ASCII.

Comparing ASCII to Unicode

The trouble with ASCII only using 7-bits (other than only representing English characters) meant people decided to use the 1-bit of extra space in whatever way they pleased. This lead to so many different codes being developed for different countries.

An example of different codes would be the computer. A letter on one computer could be a completely different letter on another computer. On some PC they could have the letter é, but on other computers sold in Israel it would be the Hebrew letter Gimel.

So, if someone from America send their résumés to Israel they would arrive as rsums [Nelson & Gailly, 1996].

The result of viewing text in one encoding using the display system of another encoding is called mojibake, a Japanese term meaning “character-mutation” [Crippen, 2010, p6]. Mojibake can be compared to words and phrases that are “lost in translation” in verbal language.

Localization

Software has to be developed or modified to be translated easily into several different languages.

The goal of this internationalization is to get a software which does not contain any language- or culture-dependent information. Also it should work in every region of the world in a way the users want.

Some components of Internationalization include:

  • User Interfaces and printed documents
  • Character classification
  • Character transliteration

  • Numerical/Monetary formats

  • Collating or sorting
  • Regular expressions
  • Encoding schemes
  • Characters vs byte processing
  • Text directionality

Language Preservation

The translations of texts and articles in library systems. Users are not only able to use their native language to search for information without having to guess at how they might have been transliterated into Latin text, but they can also search through for information from articles of a different language. This is especially useful when trying to display a record that could contain multiple different translations versions [Tull, 2002, pg 181]

Unicode can also help collect and store a multitude of different languages, preserving it for years to come. India is a country rich in languages, cultures, customs and religions. The records kept by these cultures contain 3,000 years of Indian history, and it is up to museums and libraries to properly document and preserve them.

Many websites at the early stages of the new internet tend to only create it for their native language. With the internet now becoming the go to choose for research and information, many people are paying more and more attention to internationalization [King, 2012, pg 60]. Needing to have websites that can be accessible to everyone no matter their language (ie. Government websites or Hospitals) is crucial for getting the correct information to the user with no errors at all.
